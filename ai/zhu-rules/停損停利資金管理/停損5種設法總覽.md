# 停損5種設法總覽

- **Rule ID**: R-RISK-01
- **名稱**: 停損5種設法總覽
- **分類**: 停損停利資金管理
- **原文與頁碼**: P11-C1-進場前的準備工作.md，下冊 p.309-312（「進場後可能走勢的應對措施」之「（一）停損」：停損的意義與功能、設停損5項重要原則、停損的5種設法、絕對停損5種必須停損情境）
- **解讀**: 本規則是全書停損邏輯的最高層級總覽，統整書中對「停損」這件事的完整框架，分三層：
  1. **停損的意義與功能**（p.309）：停損＝停止損失，從市場中取回主動權；功能是①控制風險、脫離不可預測的危險，②保留資金、保存反敗為勝的本錢。
  2. **設停損5項重要原則**（p.309）：①進場策略擬定時就要同時設好停損點，不能只想買不想錯的情況；②停損點設定後不得任意更改，否則等於沒設停損；③當天收盤價做多跌破（或做空突破）停損價須立即出場，須絕對遵守紀律；④為減少停損次數，應以順勢交易為主、在高勝率位置進場（呼應R-RISK-06「時機點1」）；⑤停損設定以**不超過10%**為硬性上限標準。
  3. **停損的5種具體設法**（p.309-312）：書中歸納5種可交替使用的停損設定方法，本規則庫中每一種在其他分類已有各自的具體程式化實例，此處僅作方法論分類與交叉引用，不重複定義計算細節：
     - **①K線停損法**：做多以進場當天K線最低點為停損點；做空以進場當天K線最高點為停損點。→ 具體實例見 `R-CANDLE-32 機械化多頭K線交易規則`、`R-CANDLE-33 機械化空頭K線交易規則`（K棒型態分類，並疊加7%停損上限）。
     - **②趨勢停損法**：做多時，回後買上漲的停損設在轉折波低點，盤整突破買上漲的停損設在突破點；做空時對稱設在轉折波高點／跌破點。→ 具體實例見 `R-TREND-01 轉折波定義與取點演算法`、`R-TREND-14 多頭短線選股與停損停利SOP`、`R-TREND-15 空頭短線選股與停損停利SOP`（趨勢判定分類）。
     - **③均線停損法**：以進場依據的均線作為停損，例如站上5日均線買進，停損即設在進場當日的5日均價。→ 具體實例見 `R-MA-21 均線戰法停損點設定邏輯（5%分界法）`及9種均線戰法規則（R-MA-22～R-MA-30，均線分類）。
     - **④固定比例停損法**：以買進價的2%～10%為停損點，依個人風險容忍度決定，一般可用固定5%；書中強調華爾街操盤高手絕對不容許超過10%的損失。→ 具體實例見均線分類的5%分界法（R-MA-21）與K棒型態分類的7%機械停損（R-CANDLE-32/33），皆為此方法的具體參數化版本。
     - **⑤支阻停損法**：做多時以關鍵支撐跌破為停損；做空時以關鍵壓力突破為停損。→ 具體實例見支撐壓力分類的各項支撐/壓力規則，例如 `R-SR-01 轉折高點壓力與突破後轉支撐規則`、`R-SR-02 轉折低點支撐與跌破後轉壓力規則`、`R-SR-04 盤整區上下頸線支撐壓力規則`、`R-SR-15 多頭回檔四大關鍵支撐買進訊號規則`、`R-SR-16 空頭反彈四大關鍵壓力放空訊號規則`等。
  4. **絕對停損（5種必須停損情境）**（p.312）：以下情境屬於「明顯判斷錯誤、絕對要停損」的位置，否則極易造成重大損失：①盤整區布局多單、股價跌破盤整區，絕對要停損；②多頭高檔趨勢反轉、空頭確認，多單絕對要停損；③空頭低檔趨勢反轉、多頭確認，空單絕對要停損；④股價跌幅超過10%，絕對要停損，不可再拗單；⑤逆勢單的反轉，要立刻停損。
  這5種設法並非互斥，實務上常合併使用（例如：先用K線停損法設初始停損，價格波動後改用趨勢停損法上移），本規則的價值在於提供一個「方法選擇框架＋跨分類索引」，而非重新定義已被其他分類覆蓋的計算邏輯。
- **可程式化**: 部分（5種方法各自的計算公式已在對應分類的具體規則中程式化；本規則作為總覽與方法選擇邏輯，屬於「元規則」性質，可程式化為一個依情境選用停損方法的路由函式，但「選用哪一種方法」本身仍需人工/策略層決策）
- **所需資料**: 依所選方法而定：進場當根K棒OHLC（K線停損）、轉折波高低點（趨勢停損）、對應均線數值（均線停損）、買進價（固定比例停損）、支撐/壓力價位（支阻停損）
- **計算公式**:
```
# 停損方法路由（元規則，實際計算見各引用規則）
FUNCTION 決定停損價(method, direction, context):
    IF method == "K線停損":
        RETURN K線停損_停損價(direction, context)          # 見 R-CANDLE-32 / R-CANDLE-33
    ELIF method == "趨勢停損":
        RETURN 趨勢停損_停損價(direction, context)          # 見 R-TREND-01 / R-TREND-14 / R-TREND-15
    ELIF method == "均線停損":
        RETURN 均線停損_停損價(direction, context)          # 見 R-MA-21 及 R-MA-22~R-MA-30
    ELIF method == "固定比例停損":
        pct = clamp(context.pct, 2%, 10%)                  # 預設5%，不超過10%硬上限
        RETURN context.entry_price * (1 - pct) if direction == "多" else context.entry_price * (1 + pct)
    ELIF method == "支阻停損":
        RETURN 支阻停損_停損價(direction, context)          # 見 R-SR-01/02/04/15/16 等

# 停損執行紀律（5項原則的程式化表達）
IF Close[t] < stop_loss (做多) OR Close[t] > stop_loss (做空):
    立即出場 = True    # 收盤價觸價即出場，停損點不得中途更改

# 絕對停損（5種必須停損情境，任一命中則強制出場，優先權高於一般停損邏輯）
IF (盤整區布局多單 AND 跌破盤整區)
   OR (多頭高檔轉空頭確認 AND 持有多單)
   OR (空頭低檔轉多頭確認 AND 持有空單)
   OR (浮虧幅度 >= 10%)
   OR (逆勢單 AND 趨勢反轉):
    強制平倉 = True   # 不可再拗單/攤平
```
- **參數**: 停損硬性上限＝10%（書中明確，任何方法都不可突破）；固定比例停損預設值＝5%（可調範圍2%～10%）
- **可回測**: 是（各引用規則本身可回測；本規則作為總覽索引亦可用於「跨方法比較停損效果」的回測研究）
- **信心**: 93/100 高（5項原則、5種設法、5種絕對停損情境，書中皆有明確條列與頁碼依據）
