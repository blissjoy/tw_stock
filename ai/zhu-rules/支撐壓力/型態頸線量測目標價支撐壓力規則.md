# 型態頸線量測目標價支撐壓力規則

- **Rule ID**: R-SR-12
- **名稱**: 型態頸線量測目標價支撐壓力規則
- **分類**: 支撐壓力
- **原文與頁碼**: P06-C3-走勢圖上看不見的支撐與壓力.md（下冊 p.90-93，「型態目標價的支撐與壓力」）
- **解讀**: 支撐壓力來源是價格型態（頭肩頂/底、圓弧頂/底、N字/倒N字、三重頂/底）完成突破或跌破頸線後，依「量出法」（型態高度對稱投射）估算出的目標價，屬於「看不見、需計算」的支撐壓力類型。取這個價位的具體做法：
  - **底部型態完成後（上漲目標價＝日後壓力）**：適用型態為頭肩底、圓弧底、N字底、三重底。先取型態的最低點價位與頸線價位，型態高度 = 頸線價 − 型態最低點價；股價收盤突破頸線（突破確認買點，書中示意圖同時標示需伴隨大量）後，上漲目標價 = 頸線價 + 型態高度 = 2×頸線價 − 型態最低點價。股價上漲到此目標價，該處轉為壓力。
  - **頭部型態完成後（下跌目標價＝日後支撐）**：適用型態為頭肩頂、圓弧頂、倒N字頭（書中原文印刷為「倒N字底」，依全書「上漲用N字底、下跌用N字頭」的對稱邏輯判斷應為排版誤植，整理時如實保留原文並註記存疑）、三重頂。先取型態的最高點價位與頸線價位，型態高度 = 型態最高點價 − 頸線價；股價收盤跌破頸線（跌破確認做空點）後，下跌目標價 = 頸線價 − 型態高度 = 2×頸線價 − 型態最高點價。股價下跌到此目標價，該處轉為支撐。
  - 目標價公式為依書中示意圖箭頭對稱長度倒推所得（書中該頁未給文字算式），整理時標註為推斷，屬於作者慣用的「頸線對稱量測」原理。
- **可程式化**: 部分（頸線價位、型態最高/最低點需先由型態辨識模組給出；本規則庫目前未見獨立的頭肩頂/底、圓弧頂/底、N字、三重頂/底型態辨識規則，須另行建置型態辨識前置模組後才能完整程式化）
- **所需資料**: 已辨識型態的頸線價位、型態最高點或最低點價位、突破/跌破當日成交量
- **計算公式**:
```
# 底部型態（頭肩底/圓弧底/N字底/三重底）完成後
function bullish_pattern_target(neckline_price, pattern_low):
    pattern_height = neckline_price - pattern_low
    target_price = neckline_price + pattern_height     # = 2*neckline_price - pattern_low
    return target_price   # 到達此價位轉為壓力

if close[t] > neckline_price and 伴隨大量:
    signal = "突破頸線確認買點，目標價（未來壓力） = bullish_pattern_target(neckline_price, pattern_low)"

# 頭部型態（頭肩頂/圓弧頂/倒N字頭/三重頂）完成後
function bearish_pattern_target(neckline_price, pattern_high):
    pattern_height = pattern_high - neckline_price
    target_price = neckline_price - pattern_height     # = 2*neckline_price - pattern_high
    return target_price   # 到達此價位轉為支撐

if close[t] < neckline_price:
    signal = "跌破頸線確認做空點，目標價（未來支撐） = bearish_pattern_target(neckline_price, pattern_high)"
```
- **參數**: 無額外可調參數；準確度完全取決於型態辨識模組給出的頸線與極值點是否正確
- **可回測**: 部分（目標價公式明確可回測，但前提的型態自動辨識書中未給演算法，屬推斷公式，需先驗證型態辨識邏輯的正確性）
- **信心**: 58/100 中（公式為書中示意圖比例推斷而非書中文字明載算式，且依賴型態辨識前置模組）
