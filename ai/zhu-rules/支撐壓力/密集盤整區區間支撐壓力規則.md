# 密集盤整區區間支撐壓力規則

- **Rule ID**: R-SR-03
- **名稱**: 密集盤整區區間支撐壓力規則
- **分類**: 支撐壓力
- **原文與頁碼**: P06-C2-走勢圖中的支撐壓力位置.md（下冊 p.039-044，6-2節「密集盤整區的支撐與壓力」）
- **解讀**: 支撐壓力來源是整個盤整區間（股價在盤整期間來回震盪形成的高低點區塊），與轉折高低點不同，取的不是單一價位而是**整個區塊**。取這個價位的具體做法：先框出盤整區的上緣（區間內最高點）與下緣（區間內最低點）。股價突破盤整區上漲後回檔，若跌回此區間，**整個區塊**都具支撐作用，其中越靠近上緣，支撐力道越強，越靠近下緣，力道越弱（遞減）。股價跌破盤整區下跌後反彈，若漲回此區間，整個區塊都具壓力作用，越靠近下緣壓力越強，越靠近上緣力道越弱。盤整整理的時間越長、期間累積成交量越大，日後這個區塊的支撐/壓力力道越強。轉換條件：股價收盤跌破盤整區下沿，代表支撐被完全跌破，整個盤整區反轉為壓力區；反之收盤突破上沿，整個盤整區反轉為支撐區。
- **可程式化**: 部分（盤整區的自動框定需先定義偵測邏輯，書中未給精確門檻，須自行設定區間收斂條件）
- **所需資料**: 每日OHLC、成交量、盤整區偵測所需的滾動窗口
- **計算公式**:
```
function detect_consolidation_zone(bars, window, range_pct_threshold):
    # 書中未給精確門檻，以下為建議偵測邏輯
    for each rolling window of size `window`:

        zone_low  = min(Low in window)
        if (zone_high - zone_low) / zone_low <= range_pct_threshold:
            候選盤整區 = (zone_low, zone_high, 起訖日期, cumulative_volume)
    return 盤整區列表

function support_or_resistance_strength(price_in_zone, zone_low, zone_high, direction):
    # direction = "回測支撐"（自上方突破後回檔）或 "回測壓力"（自下方跌破後反彈）
    position_ratio = (price_in_zone - zone_low) / (zone_high - zone_low)  # 0=下緣, 1=上緣
    if direction == "回測支撐":
        strength = position_ratio          # 越接近上緣（比例越高）支撐越強
    elif direction == "回測壓力":
        strength = 1 - position_ratio       # 越接近下緣（比例越低）壓力越強
    strength_bonus = f(整理時間長度, 期間累積成交量)   # 時間越長、量越大，力道加成，書中未給量化係數

function zone_role_transition(close_price, zone_low, zone_high):
    if close_price < zone_low:
        新角色 = "整個盤整區轉為壓力區"
    elif close_price > zone_high:
        新角色 = "整個盤整區轉為支撐區"
```
- **參數**: window（盤整區偵測窗口天數，書中未給）、range_pct_threshold（區間收斂百分比門檻，書中未給，需自行校準）
- **實作決定（2026-07-22，工程補充）**：本規則庫在R-CANDLE-04已實作`detect_consolidation()`（見`src/indicators/consolidation.py`），用「逐日動態擴張、區間不再擴大即視為橫盤延續」的演算法框定盤整區，**不需要額外的range_pct_threshold百分比參數**——這比另訂一個外部百分比門檻更嚴謹（不需校準、不因參數選擇不同而得出不同結果），故R-SR-03直接沿用此演算法，不另外設計新的偵測邏輯。查證外部資料未找到「盤整區偵測」的公認統一window/threshold慣例（多數技術分析資料僅描述概念，未給量化標準），故此處以「重用既有規則的無參數演算法」取代「引入一個需要外部佐證的新參數」，判斷邏輯上更站得住腳。
