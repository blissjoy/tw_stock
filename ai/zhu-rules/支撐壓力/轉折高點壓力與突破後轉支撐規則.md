# 轉折高點壓力與突破後轉支撐規則

- **Rule ID**: R-SR-01
- **名稱**: 轉折高點壓力與突破後轉支撐規則
- **分類**: 支撐壓力
- **原文與頁碼**: P06-C2-走勢圖中的支撐壓力位置.md（下冊 p.031-034，6-2節「轉折波高點（頭）的支撐與壓力」）
- **解讀**: 支撐壓力來源是轉折波的最高點（頭），取點方法沿用[[轉折波定義與取點演算法]]，不重新定義轉折點本身。取這個價位的具體做法：取轉折高點（頭）當根K線的最高價，畫一條水平線作為壓力／支撐參考。判斷邏輯：只要股價還在該頭部**下方**，上漲接近此價位時，頭部即形成壓力（代表當初套牢賣壓可能出籠）；一旦股價**收盤突破**該頭部，此後股價回檔跌回頭部價位，頭部反過來變成支撐。同一個轉折高點，其「壓力」或「支撐」身分完全由股價目前相對其位置決定，並非固定屬性。無論被判定為壓力還是支撐，都不是絕對有效——是否真的止漲/止跌，須以當日K棒反應＋次日確認（見[[遇壓遇撐4法則支撐壓力有效性確認規則]]）為準，不可僅憑股價接近頭部價位就直接判定。
- **可程式化**: 是
- **所需資料**: 依[[轉折波定義與取點演算法]]取得的轉折高點（頭）價位序列（含日期）、當前股價OHLC
- **計算公式**:
```
turning_points = compute_turning_points(bars, N)   # 見 [[轉折波定義與取點演算法]]
heads = [p for p in turning_points if p.type == "head"]

function classify_head_role(head, current_price):
    if current_price < head.price:
        role = "壓力"      # 股價尚未突破頭部
    elif current_price >= head.price and 曾經收盤突破過 head.price:
        role = "支撐"      # 突破後回測
    return role

# 進場/確認仍需套用 [[遇壓遇撐4法則支撐壓力有效性確認規則]]：
# 壓力情境：股價接近head.price當天K棒型態 + 次日方向確認
# 支撐情境（突破後回測）：股價回檔接近head.price當天K棒型態 + 次日方向確認
```
- **參數**: 繼承轉折波取點演算法的均線天期N（5/10/20日短中長線版本）
- **可回測**: 是
- **信心**: 89/100 高
