# 頭部底部大量關卡支撐壓力轉換規則

- **Rule ID**: R-SR-05
- **名稱**: 頭部底部大量關卡支撐壓力轉換規則
- **分類**: 支撐壓力
- **原文與頁碼**: P06-C2-走勢圖中的支撐壓力位置.md（下冊 p.047-052，6-2節「空頭壓力與多頭支撐的區間變化（一）空轉多」「（二）多轉空」）
- **解讀**: 描述空頭／多頭趨勢反轉確認過程中，頭部或底部區間內「成交量最大的關鍵K線」所形成的支撐壓力關卡，與一般轉折高低點（R-SR-01/02）不同之處在於：取點依據是**該區間內成交量最大的價位**，而非單純最高/最低點。
  - **空轉多方向**：股價向下跌破盤整、空頭確認後，頭部區間通常有**2個高點大量**（成交量異常放大的K線），這2個大量高點的價位形成套牢壓力關卡；此後股價下跌一波段打底，打底期間出現的大量價位形成進貨支撐。股價向上突破頭部2個大量高點所形成的壓力關卡後，容易展開大漲一波，是做多買點，原壓力關卡轉為之後回測的支撐。
  - **多轉空方向**（結構對稱、方向相反）：股價向上突破盤整、多頭確認後，底部區間通常有**2個低點大量**，形成進貨支撐關卡；此後股價上漲一波段又轉為盤整，盤整期間出現的大量價位形成套牢壓力。股價向下跌破底部2個大量低點所形成的支撐關卡後，容易展開大跌一波，是做空賣點，原支撐關卡轉為之後反彈的壓力。
- **可程式化**: 部分（需先由趨勢反轉確認模組界定「頭部區間」「底部區間」的起訖範圍，本規則庫的[[頭頭高底底高多頭趨勢判定]]／[[頭頭低底底低空頭趨勢判定]]可作為前置輸入）
- **所需資料**: 已界定的頭部／底部區間內每日OHLV（含成交量）
- **計算公式**:
```
function find_key_volume_levels(zone_bars, zone_type, top_n=2):
    # zone_type = "頭部"：取區間內成交量最大的 top_n 根K線，取其「最高價」為關鍵壓力關卡
    # zone_type = "底部"：取區間內成交量最大的 top_n 根K線，取其「最低價」為關鍵支撐關卡
    sorted_bars = sort(zone_bars, key=Volume, descending=True)[:top_n]
    if zone_type == "頭部":
        key_level = max(bar.High for bar in sorted_bars)
    elif zone_type == "底部":
        key_level = min(bar.Low for bar in sorted_bars)
    return key_level

# 空轉多：
頭部關鍵壓力 = find_key_volume_levels(頭部區間, "頭部", top_n=2)
if 收盤價 > 頭部關鍵壓力 且 確認突破:
    signal = "突破頭部大量壓力關卡，做多買點；原壓力轉為支撐"

# 多轉空：
底部關鍵支撐 = find_key_volume_levels(底部區間, "底部", top_n=2)
if 收盤價 < 底部關鍵支撐 且 確認跌破:
    signal = "跌破底部大量支撐關卡，做空賣點；原支撐轉為壓力"
```
- **參數**: top_n（取關鍵大量K線的數量，書中固定為2，可調整）
- **可回測**: 是
- **信心**: 62/100 中（關卡取點邏輯明確，但「頭部/底部區間」邊界的自動框定需搭配趨勢判定模組，書中未給精確演算法）
