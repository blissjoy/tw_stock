# 量價因果關係與四維度綜合判讀原則

- **Rule ID**: R-VOLPRICE-02
- **名稱**: 量價因果關係與四維度綜合判讀原則
- **分類**: 量價
- **原文與頁碼**: `P07-C1-對成交量的新認識.md`（下冊 p.099-101）
- **解讀**: 本規則是量價分析的方法論前提，用於約束所有量能規則的使用方式，避免誤用單一量能訊號：
  1. **成交量是零和數字**：當日成交量同時等於買進與賣出的數量，本身不具方向性，不能單憑量增/量縮直接推斷多空，須搭配當天或後續價格變化才能判斷該量偏多方還是偏空方意涵。
  2. **量與價非絕對因果**：「有量才有價」與「有價才有量」兩種市場說法皆可能成立，無論多頭、空頭或盤整趨勢都可能出現任一種價量組合（量增上漲、量縮上漲、量增下跌、量縮下跌）；不應僅以量增/量縮單一條件判斷是否續漲/續跌。
  3. **股價才是市場重心，量只是手段**：唯有股價漲跌才能讓做多/做空雙方賺到價差；即使當天暴出大量，只要股價不漲甚至下跌，做多者依然無法獲利。分析成交量異常時，應先問「這個量是否服務於拉抬/壓低股價」，而非把放量本身當作獨立的多空訊號。
  4. **主力對敲風險**：底部區出現爆量突破時，需留意是否存在主力對敲（同一主力不同帳戶間自我成交）的可能，不能單純以「大量＝買盤積極」解讀，應搭配後續股價是否延續上漲來驗證該量是否為有效攻擊量。
  5. **精準研判成交量需結合4個維度**（缺一容易失誤）：①趨勢方向（多頭／空頭／盤整）；②位置（起漲、起跌、行進中、突破、跌破、壓力下沿、支撐上沿）；③股價漲跌（成交量增減的「當天或次日」，股價是上漲或下跌）；④成交量的增減幅度。
  6. **量能僅是影響股價的因素之一**：與經濟面、基本面、消息面、技術面、指標面、政治面、心理面、籌碼面並列，程式化規則設計時不應把量能指標當成唯一或決定性訊號。
- **可程式化**: 部分（第4、5點的「4維度框架」可作為所有量價規則的必要輸入變數，屬於可程式化的規則骨架；第1、3、6點屬於方法論告誡與風險提示，難以直接轉為單一觸發條件，僅能作為規則設計的前提假設或訊號可信度的權重調整依據）
- **所需資料**: 收盤價、成交量、趨勢位置標籤（起漲/起跌/行進中/突破/跌破/壓力/支撐，通常由 [[頭頭高底底高多頭趨勢判定]]／[[頭頭低底底低空頭趨勢判定]] 等規則提供）
- **計算公式**:
```
# 4維度綜合判讀骨架（供其他量價規則呼叫，而非獨立觸發訊號）
def evaluate_volume_signal(trend_direction, position, price_change_today_or_next_day, volume_change_pct):
    # 任何量能規則觸發前，都應同時檢查以上4個變數，缺一則訊號可信度下降
    if not (trend_direction and position and price_change_today_or_next_day is not None and volume_change_pct is not None):
        return "資訊不足，暫緩判定"
    return combine(trend_direction, position, price_change_today_or_next_day, volume_change_pct)

# 主力對敲風險標記（僅作為警示旗標，非否決訊號）
if 底部區 and 爆量突破 and (成交量集中於少數帳戶/券商分點，資料可得時):
    flag = "留意主力對敲可能，等待後續股價是否延續上漲驗證"
```
- **參數**: 無具體數字門檻，屬於判讀框架與風險提示
- **可回測**: 否（本規則為質性方法論與風險提示，非可直接觸發買賣訊號的量化條件；4維度框架可作為其他量化規則的必要輸入欄位間接參與回測）
- **信心**: 58/100 中（核心觀念與4維度框架在原文中描述明確＝高；但缺乏具體數字門檻、且主力對敲等提示屬於定性風險提醒，難以獨立驗證，故整體評為中）
