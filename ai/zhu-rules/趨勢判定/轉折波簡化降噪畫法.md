# 轉折波簡化降噪畫法（中波／K線橫盤／趨勢波）

- **Rule ID**: R-TREND-02
- **名稱**: 轉折波簡化降噪畫法（中波／K線橫盤／趨勢波）
- **分類**: 趨勢判定
- **原文與頁碼**: 上冊 p.040-045（見 P02-C1-認識轉折波.md）
- **解讀**: 標準轉折波（R-TREND-01）在盤整或橫盤階段容易產生大量鋸齒狀雜訊轉折點，不利於肉眼或程式判斷波浪大方向，因此書中提供三種「後處理／降噪」畫法，皆建立在標準轉折波之上：(1) **中波**：適用於行情進入盤整時，若盤整後反轉（頭部盤整反轉向下、底部盤整反轉向上），只取盤整區間內最高或最低的1點；若盤整後延續原趨勢方向（續勢盤整），則取盤整區間的高、低點各2個。(2) **K線橫盤**：適用於股價在5日均線上下頻繁穿梭、無明顯方向時，反轉向下只取1個代表性高點，反轉向上只取1個代表性低點，避免產生過多無意義的鋸齒轉折。(3) **趨勢波**：最粗略的畫法，只在「多頭回檔跌破前低」或「空頭反彈突破前高」這種確立趨勢改變的關鍵事件發生時才取轉折點連線，其餘的盤整、假突破一律忽略不標記。三者都不是取代標準轉折波，而是依情境對其做簡化處理。
- **可程式化**: 部分（中波與K線橫盤所依賴的「盤整區間」「橫盤區間」邊界，書中僅用圖例示意，未給出量化門檻，需自行定義例如天數、幅度百分比或穿越次數；趨勢波邏輯本身明確可完全程式化）
- **所需資料**: 標準轉折波頭/底序列（R-TREND-01輸出）、K線OHLC、5日均線、盤整/橫盤區間邊界（需自訂判定規則）
- **計算公式**:
```
# (1) 中波：盤整簡化
if 偵測到盤整區間(依自訂規則，例如連續多個轉折高低點皆落在一個窄幅區間內):
    if 盤整結束後方向 != 盤整前原趨勢方向:      # 反轉
        if 原趨勢為多頭(頭部盤整反轉向下):
            取盤整區間內「最高1點」為中波轉折點
        if 原趨勢為空頭(底部盤整反轉向上):
            取盤整區間內「最低1點」為中波轉折點
    else:                                        # 續勢盤整
        取盤整區間內「最高2點、最低2點」為中波轉折點

# (2) K線橫盤簡化
if 股價在MA5上下頻繁穿越(依自訂門檻，例如N日內穿越MA5次數 >= X次):
    反轉向下 -> 取代表性「高點」1個
    反轉向上 -> 取代表性「低點」1個

# (3) 趨勢波：只抓關鍵趨勢轉折
prev_confirmed_high = None
prev_confirmed_low = None
for each bar (在多頭趨勢中):
    if Close[t] < prev_confirmed_low:          # 多頭回檔跌破前低
        趨勢波取 prev_confirmed_high 為頭部轉折點
for each bar (在空頭趨勢中):
    if Close[t] > prev_confirmed_high:          # 空頭反彈突破前高
        趨勢波取 prev_confirmed_low 為底部轉折點
```
- **參數**: 盤整/橫盤判定門檻（書中未給明確數字，需自訂：例如穿越均線次數閾值、區間寬度百分比、盤整最短天數）
- **可回測**: 部分（趨勢波規則明確可直接回測；中波與K線橫盤需先自訂盤整/橫盤邊界參數才能回測）
- **信心**: 60/100 中（取到區間後的高低點運算邏輯清楚，但「什麼樣的走勢算盤整區間」「什麼算橫盤」書中用圖例示意而非量化門檻，需工程判斷補完）
