# 上漲行進缺口與逃逸缺口量測規則

- **Rule ID**: R-GAP-10
- **名稱**: 上漲行進缺口與逃逸缺口量測規則
- **分類**: 缺口
- **原文與頁碼**: P09-C2-向上突破缺口的研究.md（下冊 p.217，圖表9-2-15、9-2-16）
- **解讀**: 發生在多頭打底完成、趨勢已形成**之後**的上漲過程中（已脫離[[打底完成向上突破缺口規則]]的起漲階段，行進於主升段內部）的跳空缺口。判斷條件：①只要缺口不回補，即代表強勢上漲持續，股價續漲機率高。②若缺口在幾天內就被回補，代表缺口失去攻擊功能；但若多頭架構（均線、高低點結構，見[[頭頭高底底高多頭趨勢判定]]）未改變，仍維持多頭趨勢，只是**趨勢轉弱**（非反轉訊號，是強度降級訊號，呼應[[向上跳空缺口四層價位分級支撐規則]]的降級邏輯）。特殊型態：股價上漲一段之後，於**關鍵位置**再度出現向上跳空並強力延續上漲，此中繼型缺口稱為「逃逸缺口」或「量測缺口」。量測法則（經驗值）：逃逸缺口出現後，行情大約還有「與缺口前面那一段上漲幅度大致相等」的漲幅（書中強調僅為經驗值，非百分之百準確）。支撐位置：缺口下緣為短期支撐（「缺口支撐」），股價回測不破缺口即維持強勢；跌破則轉弱訊號。
- **可程式化**: 是
- **所需資料**: 已確認的多頭趨勢狀態（[[頭頭高底底高多頭趨勢判定]]輸出）、缺口事件、缺口前一段上漲的起漲點與缺口位置（用於量測）、後續是否回補
- **計算公式**:
```
function detect_runaway_gap_up(gap, bars, i, trend_state, prior_leg_start_price):
    if gap.type != "up_gap" or trend_state != "多頭（已脫離起漲階段）":
        return None

    result = {"category": "上漲行進缺口", "support": gap.lower_edge}

    if not is_gap_filled(gap, bars, i, within_days=N_watch):
        result["signal"] = "缺口未回補，強勢上漲持續，續漲機率高"
    else:
        if trend_state == "多頭（架構未變）":
            result["signal"] = "缺口已回補，攻擊功能喪失，趨勢轉弱但未反轉"
        else:
            result["signal"] = "缺口已回補，且多頭架構已改變，需重新評估趨勢"

    # 逃逸缺口／量測缺口的量測法則
    if is_key_position(bars, i):   # 關鍵位置判斷（書中未給精確定義，需搭配型態/均線輔助判斷）
        prior_leg_gain = gap.lower_edge - prior_leg_start_price
        result["escape_gap"] = True
        result["measured_target"] = gap.upper_edge + prior_leg_gain   # 經驗值，非保證
        result["measured_target_note"] = "量測法則為經驗值，非百分之百準確"

    return result
```
- **參數**: N_watch = 回補觀察天數（書中僅稱「幾天內」，未給精確天數，需自行定義，建議可比照突破缺口的3天窗口）；「關鍵位置」判斷條件書中未給精確定義
- **可回測**: 是
- **信心**: 60/100 中（「是否回補決定強弱」與量測法則的方向性書中明確；但「幾天內」「關鍵位置」等用詞模糊，且量測法則書中明言僅為經驗值非精確公式，量化時需自行補上判斷門檻）
