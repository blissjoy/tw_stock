# 下跌行進缺口與逃逸缺口量測規則

- **Rule ID**: R-GAP-15
- **名稱**: 下跌行進缺口與逃逸缺口量測規則
- **分類**: 缺口
- **原文與頁碼**: P09-C3-向下跌破缺口的研究.md（下冊 p.227，圖表9-3-9）
- **解讀**: 與[[上漲行進缺口與逃逸缺口量測規則]]鏡射對稱。空頭「頭部」已完成、下跌趨勢形成之後（見[[做頭完成向下跌破缺口規則]]），在下跌過程中持續出現的向下跳空缺口，屬趨勢「行進中」的續跌缺口。判斷條件：①只要不被回補，代表強勢下跌狀況，股價會繼續下跌。②若缺口在出現後幾天內即被回補，缺口便失去下跌功能；但只要空頭架構本身沒有改變（見[[頭頭低底底低空頭趨勢判定]]），仍屬空頭趨勢，只是下跌強度轉弱（並非直接翻多，呼應[[向下跳空缺口四層價位分級壓力規則]]的降級邏輯）。③特殊子類型——逃逸缺口/量測缺口：股價下跌一段之後，在關鍵位置再出現向下跳空缺口並繼續下跌，這種中繼性質的強力向下缺口稱為逃逸缺口或量測缺口。經驗上，後續跌幅會大約等同「前一段下跌」的幅度（用缺口位置為中點，前後跌幅大致對稱），書中特別註明僅為經驗值，非百分之百準確。壓力位置：缺口上緣為短期壓力（「缺口壓力」），股價反彈不過缺口即維持弱勢；突破則轉強訊號。
- **可程式化**: 是
- **所需資料**: 已確認的空頭趨勢狀態、缺口事件、缺口前一段下跌的起跌點與缺口位置（用於量測）、後續是否回補
- **計算公式**:
```
function detect_runaway_gap_down(gap, bars, i, trend_state, prior_leg_start_price):
    if gap.type != "down_gap" or trend_state != "空頭（已脫離起跌階段）":
        return None

    result = {"category": "下跌行進缺口", "resistance": gap.upper_edge}

    if not is_gap_filled(gap, bars, i, within_days=N_watch):
        result["signal"] = "缺口未回補，強勢下跌持續，續跌機率高"
    else:
        if trend_state == "空頭（架構未變）":
            result["signal"] = "缺口已回補，下跌功能喪失，強度轉弱但未翻多"
        else:
            result["signal"] = "缺口已回補，且空頭架構已改變，需重新評估趨勢"

    if is_key_position(bars, i):
        prior_leg_loss = prior_leg_start_price - gap.upper_edge
        result["escape_gap"] = True
        result["measured_target"] = gap.lower_edge - prior_leg_loss   # 經驗值，非保證
        result["measured_target_note"] = "量測法則為經驗值，非百分之百準確"

    return result
```
- **參數**: N_watch = 回補觀察天數（書中未給精確天數）；「關鍵位置」判斷條件書中未給精確定義
- **可回測**: 是
- **信心**: 60/100 中（與上漲行進版完全對稱，方向性明確；「幾天內」「關鍵位置」等用詞模糊，量測法則書中明言僅為經驗值）
