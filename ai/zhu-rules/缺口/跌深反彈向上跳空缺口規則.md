# 跌深反彈向上跳空缺口規則

- **Rule ID**: R-GAP-06
- **名稱**: 跌深反彈向上跳空缺口規則
- **分類**: 缺口
- **原文與頁碼**: P09-C2-向上突破缺口的研究.md（下冊 p.207-208，圖表9-2-1、9-2-2）
- **解讀**: 空頭下跌走勢（可參考[[頭頭低底底低空頭趨勢判定]]）中，股價急跌後出現向上跳空的紅K棒，屬於「空頭中的反彈」，並非趨勢轉多，操作上須先當反彈處理，不可貿然視為多頭進場訊號。判斷條件：①若反彈未過前面高點，視為單純空頭反彈，不可進場做多。②若跳空強勢反彈突破月線（MA20）壓力或前面高點，回檔時有機會打底，可列入「打底觀察股」名單追蹤（後續是否形成[[打底完成向上突破缺口規則]]）。③若跳空前已出現連續3天以上中長黑K急跌或連續跌停板、且爆大量，收盤前確認當天收盤價突破前一天高點，可少量買進搶反彈，須同時設好停損停利（屬短線搶反彈，非趨勢認定）。缺口本身（跳空前一日收盤與跳空當日開盤之間的價格區間）視為短期支撐；若回檔跌破缺口下緣，反彈訊號即失效。
- **可程式化**: 是
- **所需資料**: 前置空頭趨勢判定（[[頭頭低底底低空頭趨勢判定]]輸出）、缺口事件（[[缺口基本定義與偵測規則]]）、近期波段前高、MA20、跳空前連續K棒型態與成交量、近N日均量
- **計算公式**:
```
function detect_rebound_gap(bars, i, trend_state, swing_high, ma20, avg_volume_N):
    gap = detect_gap(bars, i)
    if gap is None or gap.type != "up_gap":
        return None
    if trend_state != "空頭":
        return None   # 需先確認前置為空頭趨勢

    result = {"gap": gap, "category": "跌深反彈缺口"}

    if bars[i].Close <= swing_high:
        result["signal"] = "未過前高，僅視為空頭反彈，不可做多"
    elif bars[i].Close > swing_high or bars[i].Close > ma20:
        result["signal"] = "突破月線或前高，列入打底觀察股名單"

    # 急跌後爆量反彈搶短條件
    pre_bars = bars[i-4:i]  # 跳空前連續數日
    consecutive_black_or_limitdown = count_consecutive_long_black_or_limitdown(pre_bars)
    had_huge_volume = any(b.Volume > K_huge * avg_volume_N for b in pre_bars)
    if consecutive_black_or_limitdown >= 3 and had_huge_volume and bars[i].Close > bars[i-1].High:
        result["short_trade_signal"] = "可少量搶反彈（需設停損停利，短線操作）"

    result["support"] = gap.lower_edge   # 缺口下緣為短期支撐
    return result
```
- **參數**: K_huge = 爆大量門檻倍數（書中未給精確值，需自行校準）；連續黑K/跌停天數門檻＝3天（書中明確）
- **可回測**: 是
- **信心**: 62/100 中（型態結構與3項判斷條件書中敘述明確；但「爆大量」門檻與「打底觀察股」的後續量化追蹤方式未給精確數值，屬短線搶反彈的風險提示條款也非嚴格量化）
