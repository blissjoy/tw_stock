# 缺口成因三分類規則

- **Rule ID**: R-GAP-02
- **名稱**: 缺口成因三分類規則
- **分類**: 缺口
- **原文與頁碼**: P09-C1-建立缺口基本觀念.md（下冊 p.200-204）
- **解讀**: 缺口一旦被[[缺口基本定義與偵測規則]]偵測出來後，作者將其成因分為三大類，不同成因對應不同的訊號權重，可用於過濾/加權缺口型態規則的訊號強度：①**消息面缺口**（突發利多/利空刺激）——影響大多偏短期震盪；但若消息屬「經濟政策面」層級，影響會較深遠。②**主力發動缺口**（大額資金強力做多或做空，常搭配當日爆量/大量）——對後續走勢影響重大，是應優先關注、訊號權重最高的一類。③**除權息或增減資缺口**（公司資本額變動所致的技術性缺口）——與市場多空看法無關，意義不大，事後常見被「填權息」方式回補，不宜比照消息面或主力缺口解讀多空意涵。此分類本身不是進出場訊號，而是「訊號可信度標註」，建議在缺口事件被偵測後，先做此分類過濾，再套用本分類其餘的型態規則（如突破缺口、竭盡缺口等），除權息缺口原則上應被排除在型態規則的統計樣本之外。
- **可程式化**: 部分
- **所需資料**: 除權息／增減資公告資料（可完全程式化排除）、跳空當日與前後數日成交量（可程式化：與均量比較是否為主力大量特徵）、新聞/公告事件資料庫（可選，用於區分消息面/政策面，若無此資料則此子類無法程式化判斷）
- **計算公式**:
```
function classify_gap_cause(gap, bars, i, ex_dividend_dates, avg_volume_N):
    # 1. 除權息／增減資缺口：可完全程式化排除
    if bars[i].date in ex_dividend_dates or 當日除權息參考價已調整:
        return "除權息或增減資缺口（技術性，意義不大，建議排除於型態規則統計樣本）"

    # 2. 主力發動缺口：以跳空當日成交量是否顯著放大做代理指標
    if bars[i].Volume > K_main * avg_volume_N:
        return "疑似主力發動缺口（訊號權重最高，優先關注）"
        # K_main 書中未給精確倍數，需自行校準（建議 >=2倍 20日均量起參考）

    # 3. 消息面缺口：以量能未顯著放大、且無除權息事件時的預設分類
    return "疑似消息面缺口（預期偏短期震盪；若可比對新聞為經濟政策面消息，則放寬持續觀察天數）"
```
- **參數**: K_main = 主力大量倍數門檻（書中未給精確值，需自行校準）；除權息/增減資判定需外部公司行動資料表；消息面/政策面細分需外部新聞事件資料，若無則只能二分為「除權息」vs「非除權息」
- **參考來源查證（2026-07-22，工程補充）**：K_main查無新的外部公認慣例，沿用本規則庫R-CANDLE-23/R-TREND-13已查證的1.5~2倍均量慣例，程式化採2.0倍為預設值，非另行查證之新來源。
- **可回測**: 是（僅除權息子類可完全客觀回測；主力/消息面子類的分類準確度取決於量能門檻與新聞資料完整度）
- **信心**: 62/100 中（三分類的定性描述與權重排序在書中明確；但「消息面」與「主力發動」兩者在缺乏新聞資料時難以純用價量客觀區分，僅能用成交量放大程度做代理近似）
