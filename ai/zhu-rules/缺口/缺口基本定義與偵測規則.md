# 缺口基本定義與偵測規則

- **Rule ID**: R-GAP-01
- **名稱**: 缺口基本定義與偵測規則
- **分類**: 缺口
- **原文與頁碼**: P09-C1-建立缺口基本觀念.md（下冊 p.199，圖表9-1-1、9-1-2）
- **解讀**: 正常交易應逐檔連續成交，跳空缺口是指連續2個交易日之間，出現「完全沒有成交過的價位區間」，肉眼可在K線圖上看到明顯留白。這是全書缺口篇（Part09）所有規則的最底層輸入定義：向上跳空缺口成立條件為「當日最低價 > 前一日最高價」，缺口區間 = [前一日最高價, 當日最低價]（下沿=前一日最高價，上沿=當日最低價）；向下跳空缺口成立條件為「當日最高價 < 前一日最低價」，缺口區間 = [當日最高價, 前一日最低價]（下沿=當日最高價，上沿=前一日最低價）。缺口大小一律取兩邊界價差的絕對值。此定義以「當日最高價/最低價」對比「前一日最高價/最低價」為準（即整根K棒含影線的價格區間都不能與前一日重疊），與9-5章「隱形缺口」（僅比較開盤價與前一日收盤價，見[[隱形缺口規則]]）明確不同，屬於「標準版」缺口定義。本規則只負責缺口的精確判定與邊界標定，缺口作為支撐/壓力的基本二分處理（含永久失效規則）已在[[向上跳空缺口支撐規則]]與[[向下跳空缺口壓力規則]]（支撐壓力分類）定義；缺口更精細的四層價位分級法則在[[向上跳空缺口四層價位分級支撐規則]]與[[向下跳空缺口四層價位分級壓力規則]]（本分類）定義；缺口依出現位置的型態分類見本分類其餘各規則（跌深反彈缺口、島型反轉、突破缺口、逃逸缺口、竭盡缺口等）。
- **可程式化**: 是
- **所需資料**: 每日OHLC（開盤價、最高價、最低價、收盤價）
- **計算公式**:
```
function detect_gap(bars, i):
    prev = bars[i-1]
    curr = bars[i]

    if curr.Low > prev.High:
        return {
            "type": "up_gap",
            "lower_edge": prev.High,    # 缺口下沿
            "upper_edge": curr.Low,     # 缺口上沿
            "size": curr.Low - prev.High,
            "date": curr.date
        }

    if curr.High < prev.Low:
        return {
            "type": "down_gap",
            "lower_edge": curr.High,    # 缺口下沿
            "upper_edge": prev.Low,     # 缺口上沿
            "size": prev.Low - curr.High,
            "date": curr.date
        }

    return None   # 無缺口
```
- **參數**: 無（精確幾何定義，無需校準參數；下游型態分類與分級規則才需要量能門檻等參數）
- **可回測**: 是
- **信心**: 94/100 高（書中原文明確給出兩個方向的判斷式與數值範例，定義無歧義）
