# 向上3日2缺口要大漲規則

- **Rule ID**: R-GAP-22
- **名稱**: 向上3日2缺口要大漲規則
- **分類**: 缺口
- **原文與頁碼**: P09-C5-缺口的進階應用.md（下冊 p.250-251，圖表9-5-17～9-5-19）
- **解讀**: 連續3根交易日的K棒之間，出現**2次同方向（向上）**的跳空缺口——即第1根與第2根K棒之間有1個向上缺口，第2根與第3根K棒之間又有1個向上缺口，3天內連跳2次空，稱為「3日2缺口」，是比單一跳空缺口更密集、更強烈的主力表態訊號。多空意義**高度依賴出現位置**（四種情境）：①**打底位置**：多頭打底時，無論型態的第1支腳或第2支腳，只要連續上漲3根紅K、且3天內出現2個缺口，代表底部完成，是強勢多頭股，可鎖股做多。②**起漲位置**：多頭上漲的關鍵起漲點出現3日2缺口，是主力強力做多的宣示；只要缺口沒有被回補（見[[缺口真封口判定規則]]），股價容易大漲。③**行進位置**：行進過程中再度出現3日2缺口，只要高檔爆量後股價不下跌，容易再漲一波。④**高檔位置（警訊）**：多頭上漲到高檔才出現3日2缺口且同時爆大量，要特別注意「股價不漲或下跌」與「向上缺口是否被回補」——此時性質可能轉變為類似[[高檔末升段向上竭盡缺口規則]]的最後噴出訊號，而非續漲訊號，須依當時位置動態判讀，不能一概視為多頭訊號。
- **可程式化**: 是
- **所需資料**: 連續3根K棒OHLC、2次缺口偵測、當時股價所在位置（打底/起漲/行進/高檔，需搭配趨勢與相對高低位置判定）、成交量、近N日均量、次日K棒型態（用於高檔警訊的吊人線等變盤訊號辨識）
- **計算公式**:
```
function detect_3day_2gap_up(bars, i):
    # bars[i-2], bars[i-1], bars[i] 為連續3根K棒
    gap1 = detect_gap(bars, i-1)   # bars[i-2] -> bars[i-1]
    gap2 = detect_gap(bars, i)     # bars[i-1] -> bars[i]
    if gap1 is None or gap1.type != "up_gap":
        return None
    if gap2 is None or gap2.type != "up_gap":
        return None
    if not (is_red_candle(bars[i-2]) and is_red_candle(bars[i-1]) and is_red_candle(bars[i])):
        return None   # 需連續3根紅K

    result = {"category": "向上3日2缺口", "gaps": [gap1, gap2]}
    position = classify_position(bars, i)   # "打底" / "起漲" / "行進" / "高檔"

    if position in ("打底", "起漲"):
        result["signal"] = "強力續漲訊號，可鎖股做多；只要缺口不被回補，股價容易大漲"
    elif position == "行進":
        result["signal"] = "若高檔爆量後不下跌，容易再漲一波"
    elif position == "高檔":
        had_huge_volume = bars[i].Volume > K_huge * avg_volume_N(bars, i)
        if had_huge_volume:
            result["warning"] = "高檔爆量3日2缺口，注意股價不漲或下跌、缺口是否遭回補，可能轉為竭盡缺口式反轉警訊"
            if is_hanging_man(bars[i+1]):
                result["warning"] += "；次日出現吊人線，變盤警訊加強"

    return result
```
- **參數**: K_huge = 爆大量門檻倍數（書中未給精確值）；`classify_position` 依賴上游趨勢/型態判定模組
- **可回測**: 是
- **信心**: 77/100 中高（3日2缺口的精確結構定義明確；四種位置的多空意涵書中皆有清楚文字與圖例對照，尤其高檔警訊有具體案例〔次日大量吊人線〕佐證；但「打底/起漲/行進/高檔」位置分類本身需依賴其他上游規則的判定準確度）
