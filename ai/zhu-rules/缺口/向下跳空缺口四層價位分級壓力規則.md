# 向下跳空缺口四層價位分級壓力規則

- **Rule ID**: R-GAP-04
- **名稱**: 向下跳空缺口四層價位分級壓力規則
- **分類**: 缺口
- **原文與頁碼**: P09-C4-缺口成交量支撐與壓力.md（下冊 p.237-239，圖表9-4-10～9-4-12）
- **解讀**: 與[[向上跳空缺口四層價位分級支撐規則]]完全鏡射對稱，補完[[向下跳空缺口壓力規則]]（支撐壓力分類）未定義的精細分級。向下跳空缺口形成壓力時，以缺口下跌前、後兩根黑K線的高低點，切出由強到弱4個價位：
  - **下底（最強）** = 缺口下跌後那根黑K線的低點
  - **下沿（次強）** = 缺口下跌後那根黑K線的高點（＝缺口下邊界，等同`lower_edge`）
  - **上沿（較弱）** = 缺口下跌前那根黑K線的低點（＝缺口上邊界，等同`upper_edge`）
  - **上高（最弱／最後防線）** = 缺口下跌前那根黑K線的高點

  適用前提同樣是**放量**的向下跳空缺口（放量代表強力下跌，日後反彈時這幾個價位才具壓力意義）。

  壓力效力同樣階梯式衰減：
  1. 收盤突破「下底」→ 空頭氣勢轉弱（第一道警訊，仍屬空頭）。
  2. 收盤突破「下沿」→ 股價進入缺口區間，容易多空拉鋸；若缺口被完全回補（見[[缺口真封口判定規則]]），代表原本「強力下跌」缺口降級為「一般空頭」，**尚未翻多**（回補是「降級」不是「失效」）。
  3. 收盤突破「上沿」→ 突破缺口且進入下跌前那根黑K線實體內，空頭氣勢**完全轉弱**，須提防下一道「上高」不可再破。
  4. 收盤突破「上高」→ 突破下跌前黑K線實體最高點，判定為**空多易位**，趨勢正式轉由多方主導。
- **可程式化**: 是
- **所需資料**: 缺口事件（[[缺口基本定義與偵測規則]]輸出）、缺口前一根K棒（K_before）與缺口當根K棒（K_after）的OHLC、跳空當日成交量、近N日均量、後續逐日收盤價
- **計算公式**:
```
function build_down_gap_tiers(gap, K_before, K_after, avg_volume_N):
    if K_after.Volume < K_vol * avg_volume_N:
        return None   # 未放量，不適用四層分級法

    tiers = {
        "下底": K_after.Low,    # 最強
        "下沿": K_after.High,   # 次強，= gap.lower_edge
        "上沿": K_before.Low,   # 較弱，= gap.upper_edge
        "上高": K_before.High   # 最弱／最後防線
    }
    tiers["state"] = "強力空頭"   # 初始狀態：四層皆未破
    return tiers

function update_down_gap_state(tiers, close_t, is_true_fill):
    # is_true_fill: 由 缺口真封口判定規則 判斷缺口是否已被真封口
    if close_t > tiers["上高"]:
        tiers["state"] = "空多易位（正式翻多）"
    elif close_t > tiers["上沿"]:
        tiers["state"] = "氣勢完全轉弱（提防上高不可再破）"
    elif close_t > tiers["下沿"]:
        if is_true_fill:
            tiers["state"] = "降級為一般空頭（缺口已真封口，非強力空頭，但未翻多）"
        else:
            tiers["state"] = "缺口內拉鋸震盪"
    elif close_t > tiers["下底"]:
        tiers["state"] = "空頭轉弱警訊（仍屬空頭）"
    else:
        tiers["state"] = "強力空頭（四層皆未破）"
    return tiers["state"]
```
- **參數**: K_vol = 放量門檻倍數（書中未給精確值，建議預設1.5～2倍N日均量）；N = 均量計算天數（預設5或20日）
- **可回測**: 是
