# 低檔島型反轉規則

- **Rule ID**: R-GAP-07
- **名稱**: 低檔島型反轉規則
- **分類**: 缺口
- **原文與頁碼**: P09-C2-向上突破缺口的研究.md（下冊 p.209-212，圖表9-2-3～9-2-10）
- **解讀**: 股價下跌途中先向下跳空，接著低檔止跌／盤整數天，隨後立即以向上跳空方式上漲，整體圖形左右各一個缺口，中間止跌區塊被兩缺口「架空」，形似海中孤島，故稱「島型反轉」。結構三要素：①向下跳空缺口 → ②低檔止跌/盤整 → ③向上跳空缺口，且兩缺口的價格區間**不重疊**（向下缺口與向上缺口彼此分開，中間K線群完全架空在外，才成立孤島視覺效果）。中間盤整天數不固定，可能僅1根K線就反轉，也可能盤整較多天；**盤整日期越多，日後行情漲幅越大**。此型態不常出現，一旦出現在低檔，視為大漲走勢前兆，空單須立刻回補。三種強勢表現子型態（皆搭配MA20上彎、大量觀察）：①低檔反彈型（反轉後測試「底部高」）；②碎步上漲型（連續小紅K緩步盤堅）；③趨勢反轉型（反轉後出現多頭確認訊號，MA20轉上，型態最強）。支撐位置：向上跳空缺口下緣即為「缺口支撐」，只要不回補此缺口，維持多方架構。
- **可程式化**: 是
- **所需資料**: 連續K棒OHLC與成交量、缺口偵測（兩次，方向相反）、MA20、後續是否出現多頭確認（[[頭頭高底底高多頭趨勢判定]]）
- **計算公式**:
```
function detect_island_reversal_bottom(bars, i_down, i_consolidation_end, i_up):
    down_gap = detect_gap(bars, i_down)          # 向下跳空
    up_gap   = detect_gap(bars, i_up)             # 向上跳空
    if down_gap is None or down_gap.type != "down_gap":
        return None
    if up_gap is None or up_gap.type != "up_gap":
        return None

    # 兩缺口區間不重疊：向下缺口的上邊界 必須低於 向上缺口的下邊界（否則不構成孤島架空）
    if not (down_gap.upper_edge < up_gap.lower_edge):
        return None

    consolidation_days = i_up - i_down - 1   # 中間盤整天數（可為0，代表僅1根K線反轉）

    result = {
        "category": "低檔島型反轉",
        "consolidation_days": consolidation_days,
        "expected_rally": "盤整天數越多，日後漲幅越大（正相關，書中未給精確函數）",
        "support": up_gap.lower_edge,
        "action": "空單立刻回補；後續觀察是否轉為低檔反彈型/碎步上漲型/趨勢反轉型"
    }

    if MA20_turns_up(bars, i_up) and bars[i_up].Volume > K_vol * avg_volume_N(bars, i_up):
        if 反彈測試底部高(bars, i_up):
            result["subtype"] = "低檔反彈型"
        elif 連續小紅K緩步盤堅(bars, i_up):
            result["subtype"] = "碎步上漲型"
        elif is_bull_confirmed(bars, i_up):  # 見 頭頭高底底高多頭趨勢判定
            result["subtype"] = "趨勢反轉型（型態最強）"

    return result
```
- **參數**: K_vol = 大量門檻倍數（書中未給精確值）；「不重疊」判定為精確幾何條件（書中未逐字寫出但由圖例可明確推論為必要條件）
- **可回測**: 是
- **信心**: 76/100 中高（三要素結構、不常出現、盤整天數與漲幅正相關、支撐位置皆有明確文字與圖例佐證；三種子型態的細部觸發條件〔如「測試底部高」「碎步上漲」的精確判斷式〕書中僅定性描述，需自行量化）
