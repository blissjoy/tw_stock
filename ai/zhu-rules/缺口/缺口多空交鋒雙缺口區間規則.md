# 缺口多空交鋒雙缺口區間規則

- **Rule ID**: R-GAP-21
- **名稱**: 缺口多空交鋒雙缺口區間規則
- **分類**: 缺口
- **原文與頁碼**: P09-C5-缺口的進階應用.md（下冊 p.249，圖表9-5-14～9-5-16）
- **解讀**: 股價在一段行進之後，上方留有一個尚未回補的缺口（形成壓力），下方也留有另一個尚未回補的缺口（形成支撐），股價被夾在兩缺口之間區間震盪，多空僵持不下，稱為「多空交鋒」。可視為[[上漲行進缺口與逃逸缺口量測規則]]／[[下跌行進缺口與逃逸缺口量測規則]]（行進缺口）與[[缺口真封口判定規則]]的組合應用——用兩個缺口的邊界框出一個明確的觀察區間，等待其中一個邊界被真封口即可判定後續方向。判斷條件：①上漲行進版：上漲走勢先出現向上跳空缺口（墊出下方支撐），續漲一段後回跌又出現向下跳空缺口（墊出上方壓力），股價於是在「上有向下缺口壓力、下有向上缺口支撐」之間區間震盪。②下跌行進版（對稱）：下跌走勢先出現向下跳空缺口（形成上方壓力），續跌一段後反彈又出現向上跳空缺口（形成下方支撐），股價同樣在兩缺口之間區間震盪。③突破條件：必須有一方的缺口被真正回補（即符合[[缺口真封口判定規則]]的真封口條件），多空方向才會明朗；在兩缺口都未被真封口之前，維持區間震盪、不宜貿然單邊表態。
- **可程式化**: 是
- **所需資料**: 兩個方向相反、尚未回補的缺口事件（上方壓力缺口、下方支撐缺口）、逐日收盤價、[[缺口真封口判定規則]]的判定結果
- **計算公式**:
```
function detect_crossfire_zone(upper_gap, lower_gap, bars, i):
    # upper_gap: 上方尚未回補的缺口（形成壓力）；lower_gap: 下方尚未回補的缺口（形成支撐）
    if upper_gap.support_resistance_status == "已失效（真封口）" or lower_gap.support_resistance_status == "已失效（真封口）":
        return None   # 任一方已真封口，不再屬於交鋒區間，方向已明朗

    price_t = bars[i].Close
    if lower_gap.upper_edge_or_lower_edge <= price_t <= upper_gap.lower_edge_or_upper_edge:
        return {
            "category": "多空交鋒（雙缺口區間震盪）",
            "resistance": upper_gap_boundary_facing_price,
            "support": lower_gap_boundary_facing_price,
            "signal": "區間震盪，不宜貿然單邊表態，等待一方真封口"
        }

    if is_true_fill(upper_gap, bars[i]):
        return {"signal": "上方壓力缺口真封口，方向轉多"}
    if is_true_fill(lower_gap, bars[i]):
        return {"signal": "下方支撐缺口真封口，方向轉空"}
```
- **參數**: 無額外校準參數；依賴上游行進缺口偵測與真封口判定的參數
- **可回測**: 是
- **信心**: 62/100 中（雙缺口夾擊震盪的結構與突破條件書中明確描述並有示意圖佐證；區間內部震盪的具體操作規則〔如是否可低買高賣〕書中未展開，僅給出方向判定邏輯）
