# 缺口假封口判定規則

- **Rule ID**: R-GAP-20
- **名稱**: 缺口假封口判定規則
- **分類**: 缺口
- **原文與頁碼**: P09-C5-缺口的進階應用.md（下冊 p.246，圖表9-5-8～9-5-13）
- **解讀**: 與[[缺口真封口判定規則]]互為判別依據，是本章判讀的核心風控機制，避免把單純的「盤中觸及缺口」誤判為「缺口已被攻破」。缺口在回補過程中，K棒雖然價格上「觸及」或「跌破/突破」了缺口邊界，但因缺乏真封口所需的「大量實體反向K」條件，實際上只是短暫觸及後隨即反轉，缺口的支撐/壓力功能並未真正消失，稱為「假封口」（假回補）。判斷條件（**任一項**符合，即傾向判定為假封口）：
  - 向上跳空缺口的假封口：(a) 回補缺口的黑K留有**長下影線**（盤中雖跌入/跌破缺口，但買盤隨即進場拉回，收盤未真正棄守）；或 (b) 回補缺口當根呈現**大量紅K**（方向相反，代表買盤而非賣壓主導）；或 (c) 回補當根**出現量縮**（成交量未放大，代表跌破力道不足）。
  - 向下跳空缺口的假封口（對稱）：(a) 回補缺口的紅K留有**長上影線**；或 (b) 回補缺口當根呈現**大量黑K**；或 (c) 回補當根**出現量縮**。
  與真封口的核心判別差異：真封口＝「大量＋實體反向K＋收盤價確實越界」三條件**同時成立**；假封口＝只要三條件中**有任一項不成立**（出現長影線、或量能不足、或反向K棒），就傾向判定為假封口，缺口的支撐/壓力仍然有效。假封口出現後，原趨勢方向通常不變，甚至因騙線失敗而動能更強。
- **可程式化**: 是
- **所需資料**: 缺口事件、缺口邊界、候選回補K棒的OHLC與成交量、近N日均量
- **計算公式**:
```
function is_false_fill(gap, candidate_K, avg_volume_N):
    # 假封口 = 真封口三要素中，任一項不成立
    if is_true_fill(gap, candidate_K, avg_volume_N):
        return False

    if gap.type == "up_gap":
        touched_or_broke_lower_edge = candidate_K.Low <= gap.lower_edge
        reasons = []
        if is_black_candle(candidate_K) and has_long_lower_shadow(candidate_K):
            reasons.append("黑K留長下影線，收盤未真正棄守")
        if is_red_candle(candidate_K) and candidate_K.Volume > K_fill * avg_volume_N:
            reasons.append("大量紅K，反向K棒，買盤主導")
        if candidate_K.Volume < avg_volume_N:
            reasons.append("量縮，跌破力道不足")

    elif gap.type == "down_gap":
        touched_or_broke_upper_edge = candidate_K.High >= gap.upper_edge
        reasons = []
        if is_red_candle(candidate_K) and has_long_upper_shadow(candidate_K):
            reasons.append("紅K留長上影線，收盤未真正突破")
        if is_black_candle(candidate_K) and candidate_K.Volume > K_fill * avg_volume_N:
            reasons.append("大量黑K，反向K棒，賣壓主導")
        if candidate_K.Volume < avg_volume_N:
            reasons.append("量縮，突破力道不足")
    else:
        return False

    if len(reasons) > 0 and (touched_or_broke_lower_edge if gap.type == "up_gap" else touched_or_broke_upper_edge):
        gap.support_resistance_status = "仍然有效（假封口）"
        return True
    return False
```
- **參數**: K_fill = 大量門檻倍數（與[[缺口真封口判定規則]]共用同一套校準）；長影線判斷門檻（書中未給精確比例）
- **可回測**: 是
- **信心**: 90/100 高（三項假封口特徵〔長影線／反向大量K／量縮〕書中以條列方式明確給出，且與真封口互為對照，邏輯自洽；僅長影線與大量的精確門檻需自行校準）
