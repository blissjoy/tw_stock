# 漲多回檔向下跳空缺口規則

- **Rule ID**: R-GAP-12
- **名稱**: 漲多回檔向下跳空缺口規則
- **分類**: 缺口
- **原文與頁碼**: P09-C3-向下跌破缺口的研究.md（下冊 p.221-222，圖表9-3-1、9-3-2）
- **解讀**: 與[[跌深反彈向上跳空缺口規則]]鏡射對稱。多頭上漲趨勢（可參考[[頭頭高底底高多頭趨勢判定]]）「進行中」出現的向下跳空黑K棒，第一時間應視為「多頭回檔」的正常現象，不宜貿然做空。判斷條件：①先看回檔是否跌破前面低點——沒跌破前低，視為多頭正常回檔，不可進場做空。②若跳空強勢跌破前低點，形成「底底低」；之後若反彈不過前高，形成「頭頭低」，兩者同時成立才是反轉空頭的確認訊號（即完整符合[[頭頭低底底低空頭趨勢判定]]）。③若向下跳空前曾出現連續3天以上中長紅K急漲或連續漲停板、且爆大量，當天收盤確認跌破昨日低點，可少量搶做「急漲後回檔」的空單，以當天黑K高點為停損依據。④漲多後先出現向上跳空缺口，若幾天內股價又出現大量反轉向下的跳空缺口，稱為「島型反轉」（見[[高檔島型反轉規則]]），容易演變成強勢回檔，可少量做空。
- **可程式化**: 是
- **所需資料**: 前置多頭趨勢判定、缺口事件、近期波段前低/前高、跳空前連續K棒型態與成交量、近N日均量
- **計算公式**:
```
function detect_pullback_gap_down(bars, i, trend_state, swing_low, swing_high, avg_volume_N):
    gap = detect_gap(bars, i)
    if gap is None or gap.type != "down_gap":
        return None
    if trend_state != "多頭":
        return None

    result = {"gap": gap, "category": "漲多回檔缺口"}

    if bars[i].Close >= swing_low:
        result["signal"] = "未跌破前低，視為多頭正常回檔，不可做空"
    else:
        result["signal"] = "跌破前低，形成底底低"
        if subsequent_rebound_fails_prior_high(bars, i, swing_high):
            result["signal"] = "底底低+頭頭低同時成立，空頭反轉確認"

    pre_bars = bars[i-4:i]
    consecutive_red_or_limitup = count_consecutive_long_red_or_limitup(pre_bars)
    had_huge_volume = any(b.Volume > K_huge * avg_volume_N for b in pre_bars)
    if consecutive_red_or_limitup >= 3 and had_huge_volume and bars[i].Close < bars[i-1].Low:
        result["short_trade_signal"] = "可少量搶空單，停損設於當天黑K高點"

    return result
```
- **參數**: K_huge = 爆大量門檻倍數（書中未給精確值）；連續紅K/漲停天數門檻＝3天（書中明確）
- **可回測**: 是
