# 隱形缺口規則

- **Rule ID**: R-GAP-24
- **名稱**: 隱形缺口規則
- **分類**: 缺口
- **原文與頁碼**: P09-C5-缺口的進階應用.md（下冊 p.255-256，圖表9-5-23～9-5-27）
- **解讀**: 次日「開盤價」與前一日「收盤價」之間出現向上或向下的跳空，但到當日收盤時，股價**未將此開盤跳空回補**，這個以「開盤價 vs 前一日收盤價」為基準的跳空即稱為「隱形缺口」。隱形缺口在技術分析上，同樣具有一般缺口的支撐、壓力，或轉強、轉弱功能。與[[缺口基本定義與偵測規則]]（標準缺口）的關鍵差異：標準缺口以「當日最高價/最低價」對比「前一日最高價/最低價」，要求整根K棒（含影線）的價格區間與前一日完全不重疊，肉眼可見留白；隱形缺口僅以「當日開盤價」對比「前一日收盤價」，即使當日盤中價格區間與前一日重疊（肉眼看不出留白），只要開盤跳空、且收盤未回補此跳空，即成立（推論式定義，書中未列數學式，依文字定義與圖例推論）：
  - 向上隱形缺口：`當日開盤價 > 前一日收盤價`，且`當日收盤價未跌破前一日收盤價`。
  - 向下隱形缺口：`當日開盤價 < 前一日收盤價`，且`當日收盤價未漲回前一日收盤價`。
  隱形缺口的支撐/壓力位置以「前一日收盤價」為界，性質等同一般缺口的邊界。一段走勢中可連續出現多個隱形缺口（不限一次），常見同一走勢中交替出現多個向上、向下隱形缺口，逐級標示轉強、轉弱的節點。隱形缺口是全篇中最適合直接程式化掃描的定義，因為不依賴圖形留白的視覺判斷，改用價格數字直接比較即可全面掃描。
- **可程式化**: 是
- **所需資料**: 每日開盤價、收盤價（不需高低價）
- **計算公式**:
```
function detect_invisible_gap(bars, i):
    prev_close = bars[i-1].Close
    open_t = bars[i].Open
    close_t = bars[i].Close

    if open_t > prev_close and close_t >= prev_close:
        return {
            "type": "up_invisible_gap",
            "boundary": prev_close,   # 支撐位置＝前一日收盤價
            "date": bars[i].date
        }

    if open_t < prev_close and close_t <= prev_close:
        return {
            "type": "down_invisible_gap",
            "boundary": prev_close,   # 壓力位置＝前一日收盤價
            "date": bars[i].date
        }

    return None   # 開盤跳空但當日收盤已回補，不構成隱形缺口
```
- **參數**: 無（純數字比較定義；此為依書中文字與圖例推論之定義，書中未列數學式，屬本規則庫的合理推論版本）
- **可回測**: 是
- **信心**: 65/100 中（「開盤價 vs 前一日收盤價、且收盤未回補」的核心概念書中文字定義清楚，並有多張圖例佐證其支撐/壓力/轉強/轉弱功能；但精確判斷式〔尤其「收盤未回補」的邊界是否含等號〕為依原文推論所得，書中未給出逐字數學式，故信心評為中）
