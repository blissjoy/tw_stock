# 缺口真封口判定規則

- **Rule ID**: R-GAP-19
- **名稱**: 缺口真封口判定規則
- **分類**: 缺口
- **原文與頁碼**: P09-C5-缺口的進階應用.md（下冊 p.243-245，圖表9-5-3～9-5-7）
- **解讀**: 缺口出現後，若被後續走勢的大量長K棒確實「吃掉」缺口區間並收盤越過對側邊界，使該缺口原有的支撐/壓力性質完全消失，稱為「真封口」（真回補）。這是全篇對「缺口是否真的失效」給出的最精確判別式，取代9-2/9-3章「3天內回補即假突破/假跌破」的粗略版本，也是[[向上跳空缺口四層價位分級支撐規則]]／[[向下跳空缺口四層價位分級壓力規則]]中「缺口已真封口」狀態的判定依據，並與[[缺口假封口判定規則]]互為判別對照。真封口三要素（**須同時成立**）：
  - 向上跳空缺口的真封口：近日內股價拉回，出現①**大量**的②**實體黑K**（非長影線的假動作K），且③**收盤價**跌到缺口下沿、或跌破缺口下沿。
  - 向下跳空缺口的真封口：近日內股價回升，出現①**大量**的②**實體紅K**，且③**收盤價**漲到缺口上沿、或突破缺口上沿。
  三要素缺一即不成立真封口（改判為[[缺口假封口判定規則]]）。效果：缺口一旦被真封口，該缺口的支撐或壓力特性即完全消失，不能再拿它當支撐/壓力位使用。但須注意：真封口不必然代表趨勢反轉（書中圖例分別示範「真封口轉弱」與「真封口轉強」兩種後續走勢），須搭配當時整體多空架構判斷，不能單獨以真封口論斷反轉方向。
- **可程式化**: 是
- **所需資料**: 缺口事件、缺口邊界（上沿/下沿）、後續逐日K棒OHLC與成交量、近N日均量
- **計算公式**:
```
function is_true_fill(gap, candidate_K, avg_volume_N):
    is_large_volume = candidate_K.Volume > K_fill * avg_volume_N

    if gap.type == "up_gap":
        is_reverse_real_body = is_black_candle(candidate_K) and not has_long_shadow(candidate_K)
        close_crosses_boundary = candidate_K.Close <= gap.lower_edge
    elif gap.type == "down_gap":
        is_reverse_real_body = is_red_candle(candidate_K) and not has_long_shadow(candidate_K)
        close_crosses_boundary = candidate_K.Close >= gap.upper_edge
    else:
        return False

    if is_large_volume and is_reverse_real_body and close_crosses_boundary:
        gap.support_resistance_status = "已失效（真封口）"
        return True
    return False
```
- **參數**: K_fill = 大量門檻倍數（書中未給精確值，可與R-GAP-03/04的K_vol共用同一套校準）；`has_long_shadow` 的影線長度門檻（書中未給精確比例，需自行定義，如影線長度超過實體N倍視為長影線）
- **可回測**: 是
- **信心**: 90/100 高（三要素「大量＋實體反向K＋收盤價越界」書中以條列「重點說明」明確給出，且明確提醒真封口不必然等於反轉方向、須配合整體架構判斷；僅大量倍數與長影線門檻的精確數值需自行校準）
