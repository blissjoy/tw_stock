# 機械化多頭K線交易規則

- **Rule ID**: R-CANDLE-32
- **名稱**: 機械化多頭K線交易規則
- **分類**: K棒型態
- **原文與頁碼**: P03-C6-K線練功實戰變化.md（上冊 p.243；原書3-6節「一、K線多空操作獲利戰法」之「1. 多頭K線交易規則」）
- **解讀**: 全書K線篇中最接近可直接程式化的機械式交易規則，以「每日收盤前」為決策時點，用「前一日高/低點」作為唯一觸發基準：進場——收盤前確認股價，突破前一日高點時買進；續抱——每天收盤前檢視股價，沒有跌破前一日低點時續抱；出場——收盤前確認股價，跌破前一日低點時出場；停損——以進場當日K線股價的最低點為停損價，且不能超過7%（兩者取更早觸發、更嚴格者）。這是一套「逐日滾動比較前一根K線高低點」的順勢加碼/出場機制，不依賴特定K線型態辨識，類似海龜交易法的簡化版。書中範例（力積3553，2013/07-09）顯示即使在同一段多頭走勢中，機械規則本身也會產生數次來回進出（whipsaw），提醒此規則單獨使用時交易頻率高，需搭配趨勢過濾條件（例如僅在多頭趨勢確認後才啟用）。
- **可程式化**: 是
- **所需資料**: 每日開盤價、收盤價、最高價、最低價
- **計算公式**:
```
state = "空手"
entry_price = None
stop_loss = None

for t in 每個交易日（收盤前決策）:
    if state == "空手":
        if close[t] > high[t-1]:
            state = "持有多單"
            entry_price = close[t]
            stop_loss = max(low[t], entry_price * (1 - 0.07))   # 進場當日K棒最低點 與 7%停損 取更嚴格(更高)者
    elif state == "持有多單":
        if close[t] < stop_loss:
            state = "空手（觸發停損出場）"
        elif close[t] < low[t-1]:
            state = "空手（跌破前一日低點出場）"
        # 否則：沒有跌破前一日低點 → 續抱
```
- **參數**: 停損上限（固定7%，書中明訂）、比較基準K棒數（預設前1日）
- **可回測**: 是
- **信心**: 89/100 高
