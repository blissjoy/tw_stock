# 多空完成反轉後續強力走勢口訣

- **Rule ID**: R-STRATEGY-07
- **名稱**: 多空完成反轉後續強力走勢口訣（12口訣之7、8）
- **分類**: 綜合操作SOP
- **原文與頁碼**: `P11-C3-12個操作口訣圖形練功.md`（下冊 p.339-342）。口訣7「多頭完成反轉，要大跌」（p.339-340，圖表11-3-21～11-3-24共4例）；口訣8「空頭完成反轉，會大漲」（p.341-342，圖表11-3-25～11-3-27共3例，為口訣7的空方鏡射版本）
- **解讀**: 口訣7與口訣8直接建立在既有趨勢反轉判定規則之上，是「真反轉確認」的最後把關口訣：
  - **口訣7（多頭完成反轉，要大跌）**：原本的多頭排列／多頭格局，經型態確認正式反轉為空頭（圖上標示「多頭確認」後接續「空頭確認」，即 [[頭頭高底底高多頭趨勢判定]] 的架構轉為 [[頭頭低底底低空頭趨勢判定]]），反轉確認後通常緊接出現一波顯著跌勢（大跌），而非緩步下滑。此口訣強調「趨勢反轉確認後」初期下跌速度與力道通常偏強，適合作為多單出場、或轉為空單進場的時機參考點，而非等到跌深後才反應。書中範例11-3-21（代碼2317）與口訣6範例①同代碼，示範「該彈不彈、破低大跌」與「完成反轉、要大跌」可能是同一波跌勢中前後銜接的兩個判讀階段；範例11-3-24（代碼1310）與口訣3範例①同代碼，示範「利多不漲做頭」與「完成反轉要大跌」是同一過程的前後兩個判讀切面。
  - **口訣8（空頭完成反轉，會大漲）**：原本的空頭格局經型態確認正式反轉為多頭（「空頭確認」後接續「多頭確認」），反轉確認後通常緊接出現一波顯著漲勢，可作為空單回補、或轉為多單進場的時機參考點。範例11-3-27（代碼3498）與口訣2範例④同代碼，示範同一檔股票先出現口訣2的打底訊號（大量不跌），最終完成口訣8的反轉大漲。
  - 此口訣本質上是既有趨勢判定規則（多頭確認/空頭確認）的「狀態切換事件」加上「切換後初期動能通常強」的經驗描述，可程式化程度是12口訣中最高的一組。
- **可程式化**: 部分（趨勢反轉確認本身完全可程式化，直接複用 [[頭頭高底底高多頭趨勢判定]] 與 [[頭頭低底底低空頭趨勢判定]] 的架構切換判斷；但反轉後「大跌/大漲」的量化幅度與觀察天數書中未給精確數字，需自訂）
- **所需資料**: 轉折波頭/底序列（見 [[轉折波定義與取點演算法]]）、多頭/空頭趨勢判定結果（見 [[頭頭高底底高多頭趨勢判定]]、[[頭頭低底底低空頭趨勢判定]]）
- **計算公式**:
```
# 口訣7：多頭完成反轉，要大跌
def signal_7_bull_to_bear_reversal(t):
    if trend_state[t-1] == "多頭確認" and trend_state[t] == "空頭確認":   # 狀態切換事件
        signal = "多頭完成反轉，預期初期出現大跌"
        # 大跌幅度／觀察天數需自訂（可用近期波動度ATR倍數作為預設判斷基準）
        return signal

# 口訣8：空頭完成反轉，會大漲（口訣7的鏡射版本）
def signal_8_bear_to_bull_reversal(t):
    if trend_state[t-1] == "空頭確認" and trend_state[t] == "多頭確認":
        signal = "空頭完成反轉，預期初期出現大漲"
        return signal
```
- **參數**: 「大跌/大漲」的具體幅度與觀察天數書中未量化，建議程式化時以近期ATR或波動度倍數自訂閾值；趨勢反轉確認本身沿用 [[頭頭高底底高多頭趨勢判定]]／[[頭頭低底底低空頭趨勢判定]] 的既有參數（無新增參數）
- **可回測**: 是（趨勢反轉確認事件明確可程式化，搭配「反轉後N日報酬率」統計即可直接回測驗證此口訣的統計顯著性，不需額外自訂型態辨識模組）
- **信心**: 88/100 高（核心邏輯＝直接複用已高信心的趨勢判定規則作為狀態切換觸發點，僅在「反轉初期動能通常強」這一經驗描述上未給精確數字，整體判讀邏輯清晰、可回測性最佳）
