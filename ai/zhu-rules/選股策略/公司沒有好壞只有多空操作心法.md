# 公司沒有好壞只有多空操作心法

- **Rule ID**: R-SCREEN-02
- **名稱**: 公司沒有好壞只有多空操作心法
- **分類**: 選股策略
- **原文與頁碼**: `P10-C1-建立對股市的正確認識.md`（下冊 p.259，「技術面操作的兩句心法」）
- **解讀**: 作者提出兩句選股／操作心法，作為技術分析選股的底層原則，用以避免被基本面印象或價位高低誤導：
  1. **「公司沒有好壞，只有多空」**：不應依公司基本面優劣預設偏多或偏空；多頭時做多可以賺錢，空頭時做空一樣可以賺錢，操作方向應完全跟隨技術面趨勢判斷（見 [[頭頭高底底高多頭趨勢判定]]、[[頭頭低底底低空頭趨勢判定]]），而非公司體質好壞。書中並以金麗科（3228）為例：2023年下半年營收連續衰退、EPS為負，股價卻自2023/11/3的131元飆漲至2024/2/19的542元（漲幅413%），佐證股價可完全脫離基本面、由技術面趨勢主導。
  2. **「股價沒有高低，只有多空」**：不應以「股價已經很高」預先看空（猜頭）、也不應以「股價已經很低」預先搶進（摸底）；多頭格局中股價會不斷創新高，空頭格局中股價會不斷破新低，進出場判斷應以趨勢是否延續為準，而非價位絕對高低。
  - 這兩句心法可轉化為選股引擎的「排除性後設規則」：篩選候選股時，不應把「基本面不佳」或「股價已創新高／已經很低」直接當作排除或加碼理由，應交由趨勢判定規則決定。
- **可程式化**: 部分（心法本身是「不應排除」的負面表列，不直接產生買賣訊號；可程式化為「候選股篩選流程中不得使用基本面優劣分數與價位絕對高低作為過濾條件」的流程約束，實際多空判斷仍需交給趨勢判定類規則）
- **所需資料**: 個股趨勢狀態（多頭／空頭／盤整，見 [[頭頭高底底高多頭趨勢判定]]、[[頭頭低底底低空頭趨勢判定]]）
- **計算公式**:
```
function should_exclude_by_bias(candidate, trend_state):
    # 錯誤示範（本規則明確禁止的排除邏輯）：
    #   if candidate.fundamentals_poor: exclude()
    #   if candidate.price_at_new_high or candidate.price_at_new_low: exclude()
    # 正確做法：僅依技術面趨勢狀態決定方向
    if trend_state == "多頭":
        return "允許做多，不因基本面差或股價創新高而排除"
    elif trend_state == "空頭":
        return "允許做空，不因股價已低而排除，也不因公司體質尚可而排除做空"
    else:
        return "盤整，暫不進場，等待趨勢明朗（見 [[盤整趨勢判定與型態]]）"
```
