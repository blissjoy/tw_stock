# 上升切線畫法與線不蓋線判準

- **Rule ID**: R-LINE-01
- **名稱**: 上升切線畫法與線不蓋線判準
- **分類**: 切線軌道線
- **原文與頁碼**: P05-C1-切線的基本概念.md（上冊 p.339；原書5-1節「一、切線的正確畫法」之「1. 上升切線的畫法」，圖表5-1-1、5-1-2）
- **解讀**: 上升切線（趨勢線）是連接兩個「底底高」轉折低點所畫出、向右上方延伸的直線，作用是標示行進方向、提前預警轉折、提供支撐。畫法步驟：①取第1個轉折低點A（K棒最低價，**含下影線**，不是只看實體或收盤價）；②往右找下一個轉折低點B，且B必須比A高（「底底高」：後點價格 > 前點價格）；③用直線連接A、B並向右無限延伸，即得到1條上升切線；④畫線正確性檢查：A、B之間所有K棒（含影線）都必須落在切線上方或切線上，只要有任何一根K棒被切線切過/壓到，此切線視為畫錯，須重新選點——此即書中「線不蓋線」原則，是判斷畫線對錯的唯一書面標準。取點所需的轉折低點序列，沿用 [[轉折波定義與取點演算法]] 的輸出；「底底高」的比較邏輯與 [[頭頭高底底高多頭趨勢判定]] 所用的「底底高」概念一致，但此處只需最新2個低點依序墊高即可畫線，不要求同時滿足「頭頭高」條件（即不要求趨勢已被判定為多頭）。走勢推進、出現更新的底底高兩點時，切線需用最新一組重新畫（見 [[隨機上升次切線動態更新畫法]]）。
- **可程式化**: 是
- **所需資料**: 轉折低點序列（含下影線最低價、日期，來自 [[轉折波定義與取點演算法]]）、K棒OHLC（含最高最低價，用於線不蓋線檢查）
- **計算公式**:
```
function draw_up_tangent_line(bottoms, bars):
    # bottoms: 依時間順序排列的轉折低點序列 [(date, low_incl_wick), ...]，來自轉折波演算法
    best_line = None
    for i from len(bottoms)-2 down to 0:
        A = bottoms[i]
        B = bottoms[i+1]
        if B.low <= A.low:
            continue   # 不滿足「底底高」，跳過此配對

        line = fit_line_through(A, B)   # 通過A、B兩低點（含下影線）的直線方程 y = f(date)

        # 線不蓋線檢查：A、B之間所有K棒（含影線）不可被切線切過/壓到
        valid = True
        for bar in bars_between(A.date, B.date):
            if bar.low < line(bar.date):
                valid = False
                break

        if valid:
            best_line = line   # 取最新（最靠右）一組通過檢查的底底高兩點畫線
            break

    return best_line   # 若為 None，代表近期找不到合法的底底高配對，暫無上升切線可畫
```
- **參數**: 轉折點取點所用均線天期 N（沿用 [[轉折波定義與取點演算法]] 之 N，預設5日，10/20日可選）
- **可回測**: 是
