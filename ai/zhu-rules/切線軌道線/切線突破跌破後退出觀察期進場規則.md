# 切線突破跌破後退出觀察期進場規則

- **Rule ID**: R-LINE-13
- **名稱**: 切線突破跌破後退出觀察期進場規則
- **分類**: 切線軌道線
- **原文與頁碼**: P05-C1-切線的基本概念.md（上冊 p.361-367；原書5-1節「五、切線與趨勢變化的關係」，圖表5-1-39~5-1-51）
- **解讀**: 核心觀念：切線（上升切線被跌破，或下降切線被突破）一旦失守，走勢後續**只有3種可能方向**，不可一廂情願認定會走向某個方向，必須先進入「退出觀察期」（不立即反向進場），等出現確認訊號後才進場。3種方向：①仍走原方向（跌破上升切線後仍延續多頭，或突破下降切線後仍延續空頭）——需等待「反轉多頭確認/回測後上漲再做多」或「反彈空頭確認/回測後下跌再做空」訊號；②轉為盤整——退出觀察期後盤整整理，之後依盤整的突破做多、或跌破做空；③反轉——正式由多轉空（跌破上升切線後空頭確認）或由空轉多（突破下降切線後多頭確認）。此規則銜接 [[跌破上升切線分級與支撐壓力角色互換]] / [[突破下降切線分級與壓力支撐角色互換]]，是切線失守後「不搶進場、等方向確認」的統一進場守則，適用於上升切線被跌破與下降切線被突破兩種情境，完全對稱。
- **可程式化**: 部分
- **所需資料**: 切線突破/跌破事件（來自R-LINE-11/R-LINE-12）、後續趨勢確認訊號（[[多頭趨勢進場訊號]]、[[空頭趨勢進場訊號]]、[[盤整趨勢判定與型態]]的突破/跌破判斷）
- **計算公式**:
```
function exit_observation_period_state_machine(break_event, subsequent_bars):
    # break_event: 上升切線被跌破 或 下降切線被突破 的事件（來自R-LINE-11/R-LINE-12）
    state = "退出觀察期"   # 進入等待，不可立即反向進場

    for bar in subsequent_bars:
        if resumes_original_direction_confirmed(bar):
            # 例如：反轉多頭確認 / 反彈空頭確認 / 回測後再次上漲或下跌
            state = "原方向確認，依原方向進場"
            break
        elif enters_consolidation(bar):
            state = "轉入盤整，退出觀察期延續"
            # 盤整期間持續監控，直到盤整被突破或跌破
            for later_bar in bars_after(bar):
                if breaks_consolidation_upward(later_bar):
                    state = "盤整向上突破，依多頭方向進場"
                    break
                elif breaks_consolidation_downward(later_bar):
                    state = "盤整向下跌破，依空頭方向進場"
                    break
            break
        elif reverses_trend_confirmed(bar):
            # 例如：空頭確認（原多頭跌破上升切線後）或多頭確認（原空頭突破下降切線後）
            state = "反轉確認，依新方向進場"
            break

    return state

# 規則要點：在 break_event 發生的「當下」，狀態一律先設為「退出觀察期」，
# 嚴禁在切線剛被跌破/突破的當天就直接反向進場，必須等待以上3種確認訊號之一出現。
```
- **參數**: 無額外可調參數（依賴下游多空/盤整確認規則的參數設定）
- **可回測**: 部分（狀態機邏輯明確可程式化；但「反轉確認」「盤整確認」等下游訊號本身依賴其他規則模組，需組合使用才能完整回測）
- **信心**: 93/100 高（原書以整節篇幅＋3種情境圖表＋p.366結論段落明確陳述「退出觀察期、等方向確認再進場」的原則，並在p.367提供完整串連範例）
