# 上升切線支撐力道遞減與角度強弱判斷

- **Rule ID**: R-LINE-09
- **名稱**: 上升切線支撐力道遞減與角度強弱判斷
- **分類**: 切線軌道線
- **原文與頁碼**: P05-C1-切線的基本概念.md（上冊 p.353；原書5-1節「三、上升切線的重要功能與作用」之「1. 上升切線有支撐作用」「2. 切線角度顯示上漲的強弱」，圖表5-1-31、5-1-32）
- **解讀**: 兩條獨立的判斷依據，皆以 [[上升切線畫法與線不蓋線判準]] 或 [[隨機上升次切線動態更新畫法]] 所畫出的切線為輸入。①支撐力道遞減：股價回檔觸及同一條上升切線的次數越多，支撐有效機率越低——第1次回檔觸線支撐最強；第2次觸線支撐成功率約一半（50%）；第3次（含）以後再觸線容易被跌破。②角度顯示強弱：比較相鄰兩條（原始／次）切線的斜率，斜率越來越大（角度變陡）＝走勢越來越強；斜率越來越小（角度變平）＝走勢越來越弱。書中圖表5-1-31、5-1-32的綜合範例示範了「角度連續遞增→轉強」與「角度反轉遞減→轉弱」的完整判斷流程，並與 [[跌破上升切線分級與支撐壓力角色互換]] 銜接（角度轉弱後常伴隨跌破切線）。
- **可程式化**: 是
- **所需資料**: 上升切線歷史序列（依時間先後排列，來自R-LINE-01/05）、每次股價觸線（回檔至切線附近）的次數記錄、K棒收盤價
- **計算公式**:
```
function support_strength_by_touch_count(touch_count):
    # touch_count: 股價回檔觸及「同一條」上升切線的累計次數
    if touch_count == 1:
        return "支撐最強"
    elif touch_count == 2:
        return "支撐約50%成功率"
    else:  # touch_count >= 3
        return "容易被跌破"

function angle_strength(line_history):
    # line_history: 依時間先後排列的上升切線序列（原始切線 + 歷次隨機次切線）
    results = []
    for i from 1 to len(line_history)-1:
        slope_prev = slope(line_history[i-1])
        slope_curr = slope(line_history[i])
        if slope_curr > slope_prev:
            results.append("走勢轉強（角度變陡）")
        elif slope_curr < slope_prev:
            results.append("走勢轉弱（角度變平）")
        else:
            results.append("走勢持平")
    return results
```
- **參數**: 觸線容許誤差範圍（判斷「是否觸及切線」的價格緩衝百分比，書中未給固定數字，需自行設定，建議預設±0.5%~1%）
- **參考來源查證（2026-07-22，工程補充，本次因R-SR-06/07重用本規則邏輯而補查）**：查證外部技術分析資料，未找到「觸碰次數對應精確遞減機率（50%等）」的公認統一數字，書中50%屬原書定性描述。值得記錄的是：主流國際技術分析文獻對「觸碰次數與支撐/壓力強度」的看法與本書方向相反——普遍認為同一價位被觸碰次數越多，代表越多交易者關注該價位，支撐/壓力效果反而**增強**而非遞減（來源：[微股力ScanTrader](https://scantrader.com/article/0172eb887bcf00001503000000000000)）。此為朱老師方法論的特定觀點，本規則庫忠實依原書邏輯實作，僅在此註記兩種觀點的差異供使用者知悉，不代表外部來源證實或推翻書中數字。
- **參考來源查證2（2026-07-23，補查「觸線容許誤差」參數）**：另外查證「觸線容許誤差範圍」（判斷股價是否算觸及切線的價格緩衝百分比）這個獨立參數，外部資料僅有「畫線應觸及最多接觸點、不過度貫穿K棒實體」的定性原則（來源：[Moneta Markets趨勢線教學](https://www.monetamarkets.com/hant/analysis/how-to-draw-trendlines/)），**查無公認統一的量化緩衝百分比慣例**，維持±0.5%~1%為工程估計值，非外部佐證數字。
